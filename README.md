# transactions-api

#1 In this activity I learned how to build and deploy a REST API using Node.js, Express, MongoDB, and Heroku. I learned how to connect a MongoDB Atlas database to an application using environment variables and how to test API routes using Postman. I also learned how cloud deployment works and how to read Heroku logs to debug problems when the application crashes.
#2 The purpose of the seed.js program is to insert sample transactions into the MongoDB database automatically. It helps create test data quickly. 
#3 The most difficult part was deploying the application to Heroku and getting MongoDB Atlas connected correctly. I ran into a lot of different errors during the deployment, especially with authentication, the package-lock file, and the MongoDB connection string. It tooks me some time to figure out how Heroku config variables and logs worked and how to fix the errors step by step.
#4 I was prudent in this assignment by taking my time to test everything step by step during the setup and deployment. When I got errors with Docker, Heroku, or MongoDB Atlas, I checked the logs to understand what was wrong and fixed each problem one at a time. I also make sure my MongoDB connection string stayed in environment variable and tested all my API routes before submitting the project.
#5 I would say you need to be really careful with things like database connections, environment variables, and user input because one small mistake can mess up the whole application. During this project I had a lot of issues from small errors in the MongoDB connection string and Heroku setup, so I learned that testing everything step by step is important. I also think using Git and saving backups helps a lot because it makes fixing mistakes way easier.
